# 如何寫文章

這份說明專為「非技術背景、用 GitHub 網頁介面編輯」的寫作者準備。
你不需要安裝任何軟體，只要有瀏覽器和 GitHub 帳號就能寫文章。

---

## 一、編輯步驟（用 GitHub 網頁介面）

### 新增一篇文章

1. **打開文章資料夾**
   到 [src/content/docs/posts/articles](https://github.com/snail-step/snail-seikeijuku/tree/main/src/content/docs/posts/articles) 這一頁。

2. **點右上角的「Add file」→「Create new file」**
   （中文介面是「新增檔案」→「建立新檔案」）

3. **輸入檔名**
   在最上方的檔名欄位打上英文檔名，結尾一定要是 `.md`，例如：
   `tsmc-analysis.md`
   - 用英文小寫、單字間用 `-` 連接（這會變成網址，所以不要用中文或空格）。

4. **貼上開頭設定 + 開始寫**（下面第二段會詳細說明格式）

5. **存檔（commit）**
   寫完後拉到頁面最下面，會看到一個綠色的「Commit changes」按鈕：
   - 上方小框可以填「這次改了什麼」（例如：新增台積電分析），不填也可以。
   - 選擇「Commit directly to the main branch」。
   - 按綠色的「Commit changes」。

6. **完成！** 過幾分鐘網站會自動更新，文章就會出現在側邊欄。

### 修改已經存在的文章

1. 點進那篇 `.md` 檔。
2. 點右上角的**鉛筆圖示 ✏️**（Edit this file）。
3. 修改內容。
4. 一樣拉到最下面按「Commit changes」。

> 小提醒：改完想確認有沒有排版跑掉，可以在編輯畫面上方切到 **「Preview」** 分頁預覽。

## 二、每篇文章的開頭設定（frontmatter）

**每一篇文章最上面，都一定要有這一段**，這是給系統看的「標題卡」：

```md
---
title: 文章標題
description: 一句話說明這篇文章。
---
```

規則很簡單：

| 項目 | 說明 |
|------|------|
| `---` | 上下各一行三個橫線，**不能刪**，它是設定的開始與結束 |
| `title:` | 文章標題（會顯示在側邊欄和頁面最上方） |
| `description:` | 一句話簡介（給搜尋引擎和預覽用） |
| 冒號後面要空一格 | `title: xxx` ✅　`title:xxx` ❌ |

這段結束後（第二個 `---` 之下），才開始寫正文。

## 三、基本 Markdown 語法

Markdown 就是「用簡單符號來排版」。以下是最常用的，記這些就夠寫文章了。

### 標題（分段大小標）

```md
## 大標題
### 中標題
#### 小標題
```
`#` 越多，標題越小。文章內文請從 `##` 開始（`#` 留給系統當文章大標）。

### 粗體、斜體

```md
**這是粗體**
*這是斜體*
```

### 條列與編號

```md
- 第一點
- 第二點
  - 這是子項目（前面空兩格）

1. 步驟一
2. 步驟二
```

### 連結與圖片

```md
[顯示的文字](https://網址)

![圖片說明](圖片網址或路徑)
```
（圖片前面多一個 `!`，這是差別。）

### 引用區塊（適合放重點或引述）

```md
> 這是一段引用文字。
```

### 分隔線

```md
---
```
（單獨一行的三個橫線，會畫一條分隔線。注意：這和開頭設定用的 `---` 不同，只要不是在最上方就不會被當成設定。）

### 表格

```md
| 項目 | 說明 |
|------|------|
| A    | 內容 |
| B    | 內容 |
```

### 排版小訣竅

- **想換段落 → 空一行**。只按一次 Enter 有時不會分段，中間空一整行最保險。
- 寫完善用上方 **Preview** 分頁看效果，符號沒打對一眼就看得出來。

## 四、進階：用 template.mdx 讓文章更漂亮

當你想要「提示框、卡片、分頁、步驟」這類更精美的排版時，可以參考
[template.mdx](src/content/docs/posts/articles/template.mdx)。它就是一份**現成範例目錄**，
裡面每個效果都附上「怎麼寫」和「寫出來長怎樣」。

### 使用前要知道的兩件事

**1. 檔名要用 `.mdx`，不是 `.md`**
這些精美元件只能在 `.mdx` 檔裡用。所以建檔時檔名請取例如 `my-post.mdx`。

**2. 檔案最上方（設定的 `---` 底下）要加一行 import**
把這一行貼在文章開頭，才能使用這些元件：

```mdx
import { Aside, Badge, Card, CardGrid, Icon, Tabs, TabItem, LinkCard, LinkButton, Steps, FileTree } from '@astrojs/starlight/components';
```
（照抄即可，不用理解它的意思。用不到的元件留著也沒關係。）

### 怎麼「參考」template 來寫

流程就三步：

1. 打開 [template.mdx](src/content/docs/posts/articles/template.mdx)，找到你想要的效果（例如「提示框」）。
2. 把它「怎麼寫」框框裡的程式碼**複製**。
3. 貼到你的文章裡，把中文字換成你自己的內容。

### 最常用的幾個效果

**提示框（Aside）** — 放重點、警語最好用：
```mdx
<Aside type="tip" title="小提醒">記得看財報再進場。</Aside>
<Aside type="caution">這是注意事項。</Aside>
```
如果覺得 import 麻煩，提示框還有一個**免 import** 的寫法（`.md` `.mdx` 都能用）：
```md
:::tip[小提醒]
記得看財報再進場。
:::
```

**步驟（Steps）** — 教學、流程用：
```mdx
<Steps>
1. 第一步：開戶。
2. 第二步：入金。
3. 第三步：下單。
</Steps>
```

**連結卡片（LinkCard）** — 推薦延伸閱讀：
```mdx
<LinkCard title="台積電 26Q1 分析" href="/posts/finance-reports/tsmc-26q1/" description="法人說明會重點整理" />
```

其他還有 Badge（小標籤）、Card（卡片）、Tabs（分頁）、Icon（圖示）等，
全部都在 template.mdx 裡有範例，需要時去複製就好。

> 建議：**新手先用純 `.md` 加基本 Markdown 就很夠了**。
> 等寫順了、想讓某篇更精緻時，再回頭挑 template.mdx 裡的元件來點綴即可。
